# 創作物の保護について

**このページは、他のすべてに優先して最初に読んでください。**

---

## 結論から

Claudeは、設定を正しくすれば **あなたの原稿や創作物をAIの学習に使いません。**
以下の設定を済ませてから使い始めてください。

---

## 何が守られるのか

| 心配ごと | 回答 |
|---------|------|
| 原稿がAIの学習素材にされない？ | **されません**（学習OFFに設定済み） |
| 未発表の作品が外に漏れない？ | **漏れません**（Coworkは指定フォルダ内でのみ動作） |
| 会話の内容が残り続けない？ | **30日で自動削除**（学習OFFの場合） |
| 自分で消した会話は？ | **即座に学習対象から除外**（削除した会話は一切使われません） |

---

## 設定する人が済ませる項目（クリエイターは何もしなくてOK）

### 1. AI学習（モデルトレーニング）をOFFにする
- 設定 → プライバシー → 「Claudeの改善に使用」→ **OFF**
- これがOFFなら、会話も、読み込んだファイルの内容も、一切学習に使われません
- データ保持期間は30日（30日後に自動消去）

### 2. フォルダのアクセス範囲を限定する
- Coworkがアクセスできるのは作業フォルダ内のみ
- パソコンの他のフォルダ（写真、メール、他の書類など）には触れません
- 自分でフォルダを追加しない限り、範囲は広がりません

### 3. ファイル削除の保護
- Coworkはファイルを削除する前に **必ず許可を求めます**
- 勝手にファイルが消えることはありません

## Anthropic社（Claudeの開発元）の公式ポリシー

以下はAnthropicの公式プライバシーポリシーに基づく事実です。

**学習OFFの場合：**
- あなたの会話・ファイル内容はAI学習に **一切使われません**
- データは30日後に削除されます
- 削除した会話は、将来のモデル学習にも使われません

**Cowork固有の保護：**
- サンドボックス（隔離された仮想環境）内で動作します
- 指定したフォルダ以外にはアクセスできません
- すべての操作がログに記録され、確認できます
- いつでも操作を中断できます

**追加の保護手段（シークレットモード）：**
- 特に機密性の高い作業では「シークレットモード」が使えます
- シークレットモードの会話は、学習設定に関係なく **絶対に** 学習に使われません

## 実際の運用ルール

### 普段の執筆作業
- 通常モードでOK（学習OFFに設定済み）
- 素材フォルダに資料を入れて、いつも通り作業する

### 未発表作品・NDA案件
- 心配ならシークレットモードを使う
  （Claudeの画面左下「新しい会話」→「シークレット」を選ぶだけ）
- 作業が終わったら会話を削除してもOK

### やってはいけないこと
- 「Claudeの改善に使用」を自分でONにしない
- 知らないプラグインを自分でインストールしない
- 信頼できないサイトへのアクセスをCoworkに許可しない

## 「学習に使わない」と「覚えてくれない」は別の話です

ここが一番誤解されやすいポイントです。

**学習OFF = あなたの原稿でAIが賢くなることを禁止する**
**記憶 = あなたとの会話の中で、前に言ったことを覚えておく**

この2つは **まったく別の機能** で、別々にON/OFFできます。
推奨設定では **学習はOFF、記憶はON** です。

| | 具体例 | 推奨設定 | 動くか |
|---|---|---|:---:|
| **記憶（メモリ）** | 「前に〇〇って言ったよね」 | ON | ✅ 覚えてる |
| **指示ファイル** | 名前、仕事内容、好み | 毎回自動で読む | ✅ 毎回知ってる |
| **フォルダ内のファイル** | 素材フォルダの資料、過去の原稿 | いつでも読める | ✅ 聞けば答える |
| **AIの学習** | 原稿の内容が次のAIモデルに反映される | **OFF** | ❌ 使われない |

つまり、こういうことです：

- 「私の名前は？」→ ちゃんと答えます
- 「前に話した設定、覚えてる？」→ 覚えています
- 「素材フォルダの資料を読んで」→ 読めます
- あなたの原稿が世界中のAIを賢くする素材になる → **なりません**

**安心して使い倒してください。覚えてもらうことと、学習素材にされることは別物です。**

## 原稿の内容でBANされない？（暴力、性描写、犯罪描写）

プロのクリエイターが書くものには暴力、犯罪、恋愛の性的描写が当然含まれます。
ここを正直に説明します。

### Claudeが問題なく扱えるもの

| ジャンル | 具体例 | 対応 |
|---------|-------|:---:|
| クライムサスペンス | 殺人事件の描写、捜査シーン、犯行動機の分析 | ✅ |
| ダークドラマ | 虐待、依存症、トラウマの描写 | ✅ |
| 暴力シーン | 戦闘、喧嘩、事件現場 | ✅ |
| ラブロマンス | 恋愛感情、キス、抱擁、朝のシーン | ✅ |
| ベッドシーン（示唆） | 「二人は寝室に消えた」「朝、シーツの中で目覚める」 | ✅ |
| 心理的緊張 | 復讐、嫉妬、裏切り、精神的駆け引き | ✅ |
| 薬物・犯罪描写 | 物語の中での使用シーン、取引シーン | ✅ |
| ホラー | 恐怖描写、超常現象、心理的恐怖 | ✅ |

**Claudeは「文学的・芸術的な目的での成熟したテーマの探求」を明確に許可しています。**
プロのクリエイターの仕事の大部分は問題なく対応できます。

### Claudeが苦手なもの（正直に言います）

| 内容 | 状況 |
|------|------|
| **性的に露骨な描写** | 具体的な性行為の詳細描写は拒否されることがあります |
| **過度にグラフィックな暴力** | 残虐な拷問シーンの「詳細な動作描写」は薄められることがあります |

これは「BAN」ではなく「拒否」です。
アカウントが停止されるのではなく、「その部分は書けません」と言われるだけです。

**重要：BANされるのは「悪意ある使い方」だけです。**
例：実在の人物への暴力を計画する、児童を性的に描写する、実際の犯罪を指南する。
フィクションの創作をしている限り、アカウント停止にはなりません。

### 実際の運用（プロのクリエイターとしての現実解）

正直に言えば、プロのクリエイターにとってこの制限はほぼ問題にならないはずです。

**理由1：プロの性的描写はそもそも「示唆」が多い**
テレビドラマでも映画でも小説でも、プロの性的描写は
行為の詳細ではなく、感情・関係性・緊張感を描きます。
この範囲ならClaudeは問題なく扱えます。

**理由2：本当に露骨な描写が必要なシーンは自分で書く**
もし作品の性質上、どうしても露骨な描写が必要な場合、
そこはクリエイター自身の筆で書くべきシーンです。
AIに任せる部分ではありません。

**理由3：犯罪・暴力描写は基本的に全部OK**
殺人、誘拐、詐欺、薬物、組織犯罪、心理的な緊張と恐怖。
これらはフィクションの文脈であればClaudeは問題なく扱います。
「このトリックに矛盾がないかチェックして」も普通にできます。

### もしClaudeに断られたら

慌てずに、以下の対処法があります：

1. **言い換える** — 「このシーンの感情的な緊張感を描写して」のように依頼を変える
2. **分割する** — そのシーンだけ自分で書いて、前後のシーンをClaudeに頼む
3. **文脈を伝える** — 「プロの〇〇（職種）として、〇〇（作品名）のシーンを書いている」と明示する
4. **相談する** — 本当に困ったら設定担当者に相談（別のツールや方法がある）

## よくある不安への回答

**Q: ChatGPTで原稿を食われたって話を聞いたけど？**
A: ChatGPTはデフォルトで学習ONです。Claudeは設定でOFFにできます。
   このキットの設定ではOFFにしてあるので、同じことは起きません。

**Q: 設定を間違えて戻したらどうなる？**
A: 万が一ONになっても、ONになった後の「新しい会話」だけが対象です。
   過去の会話に遡って学習されることはありません。
   気づいたらすぐOFFに戻せば大丈夫です。

**Q: Anthropicの社員が原稿を読める？**
A: 通常の運用では読みません。ただし、利用規約違反の疑い（犯罪行為など）で
   安全性の確認が必要な場合にのみ、内容を確認する可能性があります。
   普通の創作活動が対象になることはありません。

**Q: 半年前に話したことを忘れてるんだけど？**
A: メモリには容量の限界があります。最近の会話ほどよく覚えていて、
   古い会話は薄れていきます（人間と同じです）。
   ただし「Claude Context」フォルダのファイルは毎回読み直すので、
   大事なことはそこに書いておけば絶対に忘れません。
   また、素材フォルダのファイルはいつでも読み返せます。

**Q: 100%完全に安全と言える？**
A: 100%はどんなサービスでも保証できません（メールでもクラウドでも同じです）。
   ただし、上記の設定をすべて行った状態は、
   AIサービスとして現時点で取れる最大限の保護策です。

**Q: 他のAIサービス（ChatGPT、Gemini等）より安全？**
A: 学習OFFの設定がある点、Coworkのサンドボックス構造、
   シークレットモードの存在から、
   クリエイターの創作物保護については業界最高水準と言えます。
